Efficiently computing extremal points in cylindrical coordinates

Project by Fabian Stroschke (Github)

Diese Arbeit untersucht die Verwendung von räumlichen Datenstrukturen zur effizienten Bestimmung von Extremalpunkten in Punktmengen. Die effiziente Bestimmung der Extremalpunkte wurde dann dafür genutzt, um eine effizientere Version des Gift-Wrapping Algorithmus zu entwickeln.

Auf dieser Seite befindet sich nur eine kurze Übersicht der Arbeit, für eine detaillierte Ausführung des Themas ist die Arbeit weiter unten integriert. Sollte sie nicht angezeigt werden, kann sie aus dem 'docs' Ordner der Repositories heruntergeladen werden.

Zusammenfassung

Das Finden von Extremalpunkten kann benutzt werden, um viele verschiedene Probleme zu lösen. Bekannte Beispiele dafür sind die Berechnung von konvexen Hüllen und die lineare Optimierung. Beide Probleme wurden in der Vergangenheit schon viel untersucht und wurden sehr gut optimiert. Das Finden einzelner Extremalpunkte zu beschleunigen wurde hingegen in weniger ausführlich betrachtet.

In dieser Arbeit wurde untersucht, wie Extremalpunkte effizient berechnet werden können, wenn eine Linie oder Ebene um eine Achse rotiert wird, bis sie einen Extremalpunkt trifft. Da dieses Problem für unsortierte Punkte nicht schneller als in O(n) gelöst werden kann, wurde dafür angenommen, dass die Punkte bereits in einer räumlichen Datenstruktur gegeben sind. Auf diese Weise wurde ein Algorithmus entwickelt, welcher die Eigenschaften dieser Datenstrukturen nutzt, um die Extremalpunkte effizient zu berechnen und mit minimalen Änderungen für viele verschiedene räumliche Datenstrukturen genutzt werden kann. Im schlimmsten Fall hat dieser Algorithmus eine Laufzeit von O(n), es konnte aber empirisch gezeigt werden, dass er im Durchschnitt deutlich schneller ist und sogar Laufzeiten von O(√n) erreichen kann.

Da das Bestimmen von Extremalpunkten durch Rotation nahezu identisch mit einem Teil des Gift Wrapping Algorithmus ist, wurde auf dieser Basis auch eine modifizierte Version des Gift Wrapping Algorithmus entwickelt, um zu zeigen, wofür die Berechnung von Extremalpunkten genutzt werden kann. Diese Modifizierung von Gift Wrapping ist deutlich schneller als der normale Gift-Wrapping Algorithmus, hat eine bessere empirische Laufzeit als andere neue Modifikationen von Gift Wrapping und ist in manchen Fällen sogar schneller als der Quickhull Algorithmus.

Ergebnisse

Laufzeiten (in Sekunden), um die konvexe Hülle zu berechnen, wenn die Punkte in einer Kugel verteilt sind.

Laufzeiten (in Sekunden), um die konvexe Hülle zu berechnen, wenn die Punkte auf der Oberfläche einer Kugel verteilt sind. Gift Wrapping wurde nur bis 2500k Punkte getestet.

Die Algorithmen für konvexe Hüllen haben keine optimale asymptotische Laufzeit, sind aber deutlich schneller als Gift Warpping und könnten deshalb in Anwendungen benutzt werden, wo die Benutzung von Gift Wrapping sinnvoll ist oder wo Teile der konvexen Hülle schnell berechnet werden sollen. Wenn das Verhältnis von h zu n sehr klein ist, könnten die hier vorgestellten Algorithmen auch statt Quickhull genutzt werden, um eine bessere Laufzeit zu erreichen.

Die hier entwickelten Algorithmen sind noch nicht optimal und können noch in vielen Aspekten verbessert werden. Sie sollen vor allem demonstrieren, dass Algorithmen wie Gift-Wrapping auf diese Weise optimiert werden können.

Bachelorarbeit

Gebrauchsanleitung

1. Anforderungen

Benötigt:

  • GCC / G++
  • CMAKE
  • CGAL
  • Python 3.8
    • MatplotlibCPP funktioniert mit 3.9+ möglicherweise nicht richtig
    • Sollte Python nicht richtig funktionieren/gefunden werden (z.B. "Matplotlib.pyplot not found error"), gibt es bei Github einen Branch der kein Python benötigt.

Die Algorithmen können auf Windows und Linux kompiliert und ausgeführt werden.

2. Konfiguration

include/config.h

Für die konfiguration von 'KDWrap', 'OctreeWrap' und 'Quickhull'. Beeinflusst nicht die parameter von 'demo'. Algorithmen müssen zum Anwenden neu kompiliert werden.

Attribut Name Beschreibung
"sample_size": Anzahl der generierten Punkte.
"seed": Start Seed für die Generierung der zufälligen Punkte.
"MatPlotShow": Zeigt Punkte, Startpunkte, Extremalpunkte und Konvexe Hülle (mit Quickhull erzeugt, um Extremalpunkte auf Fehler zu überprüfen) an.
Command-line Argumente für 'demo'
Attribut Name Beschreibung
"?","-?","-help": Zeigt eine Beschreibung der Befehle im Terminal an.
"-QH": Quickhull wird als Algorithmus benutzt.
"-OT": OctreeWrap wird als Algorithmus benutzt.
"-KD": KDWrap wird als Algorithmus benutzt.
"-GW": Gift-wrapping wird als Algorithmus benutzt.
"-path [string]": Pfad zu einer Datei die eine Punktwolke enthält. Siehe read_polygon_soup
"-seed [integer]": Start Seed für die Generierung der zufälligen Punkte. Kann nicht zusammen mit '-path' benutzt werden.
"-size [integer]": Anzahl der Punkte die generiert werden sollen, wenn '-seed' benutzt wird.
"-dim [x] [y] [z]": Maximale Distanz zwischen 2 Punkten entlang einer Achse. Standard Dimensionen x=100, y=100 z=100
"-shape [Option]" Eine der unten gelisteten Optionen muss für die Punktgenerierung gewählt werden.
"BoxFull": Alle Punkte werden innerhalb eines Quaders generiert.
"BoxSurface": Alle Punkte werden auf der Oberfläche eines Quaders generiert. (Es entstehen viele coplanare Punkte, welche bei Gift-wrapping, KDWrap und OctreeWrap zu Artefakten führen kann.)
"SphereFull": Alle Punkte werden innerhalb einer Kugel generiert.
"SphereSurface": Alle Punkte werden auf der Oberfläche einer Kugel generiert. (Alle Punkte sind Teil der konvexen Hülle.)

3. Ausführen

KDWrap, OctreeWarp, Quickhull:

Ausführen über IDE oder Command-line ohne Argumente. Beispiel ./KDWrap

Nach Ausführung wird die benötigte Zeit und der Seed angezeigt.

demo:

Ausführen über IDE oder Command-line mit Argumenten. Beispiel ./demo -seed 1234567 -OT -size 100000 -shape SphereFull

Nach Ausführung werden die Ergebnisse im Format "[n];[h];[t1];[t2]" angezeigt.
[n]: Anzahl der Punkte
[h]: Anzahl der Punkte auf der konvexen Hülle
[t1]: Zeit die benötigt wurde, um die Datenstruktur zu bauen, in Microsekunden
[t2]: Zeit die benötigt wurde, um die konvexe Hülle zu berechnen, in Microsekunden
